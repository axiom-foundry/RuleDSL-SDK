# Determinism comparison reports — 2026-07-11

This directory holds the **cross-platform comparison reports** for the published
determinism evidence set dated `2026-07-11`. Each report records a Windows-x64 vs
Linux-x64 byte comparison of a determinism case and is the artifact behind the
"recompute the hashes yourself" claim.

**Why this set exists:** it is the first evidence set produced by the **v1.0.2
engine** (the grammar-embed fix — see `docs/release_notes/v1.0.2.md`). Its
`output_bin_sha256` and `bytecode_sha256` values are **identical** to the
pre-fix sets dated `2026-06-21` and `2026-06-23`. That identity is the recorded
proof of the v1.0.2 compatibility claim: the defect was in *where the compiler
found its grammar*, never in *what it produced*. Unlike the earlier sets, this
one was generated by binaries that also run off the build machine.

There is one `comparison.json` per DET case:

| Report | DET case | What it compares |
| --- | --- | --- |
| `DET-001/windows-x64__linux-x64/comparison.json` | DET-001 (standard decision-output path) | Windows-x64 bundle vs Linux-x64 bundle |
| `DET-003/windows-x64__linux-x64/comparison.json` | DET-003 (deterministic error path) | Windows-x64 bundle vs Linux-x64 bundle |

Each report (schema `determinism-bundle-compare-v2`) records `status: "pass"`,
`equal_output: true`, and an `output_bin_sha256` that is **identical** on both
platforms — and identical to the 2026-06-21 / 2026-06-23 sets:

- DET-001 `output_bin_sha256` = `f3714ba24ad7a84218874bff096eab721bd67bcd5641cd45d2973fd86d55c5e1`
- DET-003 `output_bin_sha256` = `9b0dd8f9caeadd8e6ed1fabd65305041bc5d518cbf5716f54178691d88727a65`

The generating engine is recorded inside each bundle
(`environment/tool_versions.json`: `RuleDSL/1.0.2`).

The comparison reports are derived from the evidence bundles committed under

holds `inputs/`, `outputs/`, an `environment/` fingerprint, a `manifest.json`, and a
`SHA256SUMS.txt`:

| `reports/determinism_v1_0/2026-07-11/linux-x64/ci-linux/` | Linux-x64 | DET-001 |

(The Linux-only locale/timezone case DET-002 is committed alongside them under
`linux-x64/ci-linux-det002/`.)

### Note on `bundle_b_rel`

Each `comparison.json` records `bundle_b_rel` as `downloaded/linux/ci-linux[-det003]`.
That is the **compare-time source path** of bundle B: during the cross-platform compare
job the Linux bundle was downloaded into a working directory before being compared
against the Windows bundle (`bundle_a_rel`). It is an honest record of where bundle B
was read from at compare time, not a committed path. The committed equivalent of that
same Linux bundle lives here under
`reports/determinism_v1_0/2026-07-11/linux-x64/ci-linux[-det003]/` — its
`outputs/<DET>_output.bin` hashes byte-for-byte to the `output_bin_sha256` values above,
so the comparison is fully reproducible from committed artifacts.

## Recompute it yourself

All paths below are repo-relative; run from the repository root. For each DET case,
hash both platforms' decision output and confirm they equal each other **and** the
`output_bin_sha256` recorded in that case's `comparison.json`:

```sh
# DET-001 — standard decision-output path
sha256sum reports/determinism_v1_0/2026-07-11/linux-x64/ci-linux/outputs/DET-001_output.bin
sha256sum reports/determinism_v1_0/2026-07-11/windows-x64/ci-windows/outputs/DET-001_output.bin
# both MUST equal f3714ba24ad7a84218874bff096eab721bd67bcd5641cd45d2973fd86d55c5e1

# DET-003 — deterministic error path
sha256sum reports/determinism_v1_0/2026-07-11/linux-x64/ci-linux-det003/outputs/DET-003_output.bin
sha256sum reports/determinism_v1_0/2026-07-11/windows-x64/ci-windows-det003/outputs/DET-003_output.bin
# both MUST equal 9b0dd8f9caeadd8e6ed1fabd65305041bc5d518cbf5716f54178691d88727a65
```

Then verify each bundle's internal integrity against its checksum manifest:

```sh
( cd reports/determinism_v1_0/2026-07-11/linux-x64/ci-linux        && sha256sum -c SHA256SUMS.txt )
( cd reports/determinism_v1_0/2026-07-11/linux-x64/ci-linux-det003 && sha256sum -c SHA256SUMS.txt )
( cd reports/determinism_v1_0/2026-07-11/windows-x64/ci-windows        && sha256sum -c SHA256SUMS.txt )
( cd reports/determinism_v1_0/2026-07-11/windows-x64/ci-windows-det003 && sha256sum -c SHA256SUMS.txt )
```

And because the v1.0.2 release binaries actually run on your machine, the strongest
check is now available to you: download the `v1.0.2` SDK bundle, run the engine
yourself, and compare your recomputed hashes against the values above.

## Scope

This evidence proves **cross-platform** (Windows-x64 ↔ Linux-x64) byte-identical
determinism for the v1.0.2 engine, and byte-identity of v1.0.2 output with the
previously published 2026-06-21 / 2026-06-23 sets. Independent **multi-host**
reproducibility (multiple machines of the same platform) remains future work.
