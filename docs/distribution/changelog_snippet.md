# CHANGELOG Snippet Guidance

Use this snippet for repository-level changelog entries tied to a bundle release.

## Template

```markdown
## <version> - <YYYY-MM-DD>

### Distribution
- Bundle type: <Evaluation|Commercial>
- Updated manifests: MANIFEST.json, HASHES.txt, TOOLCHAIN.txt, LICENSE_STATUS.txt
- Verification flow: compile -> verify -> compatibility check -> evaluate

### Compatibility
- Compiler: <version>
- Language: <version>
- AXBC: <version>
- ABI: <level>
- Minimum engine version: <version>

### Notes
- <important operational constraints>
```

## Upgrade / Compatibility Boilerplate

```text
Before upgrading, verify HASHES.txt and confirm versions against docs/compatibility_matrix.md.
Recompile rules with the bundle compiler, run ruledslc verify on generated bytecode,
and run ax_check_bytecode_compatibility before evaluation.
```
