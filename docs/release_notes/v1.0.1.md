# RuleDSL SDK v1.0.1

> **Erratum (2026-07-11):** This note claimed the bundle is self-sufficient. The claim was wrong:
> v1.0.0/v1.0.1 binaries load the grammar from a filesystem path that only exists on the machine
> that built them, so compilation fails on every other machine. The verification behind the claim
> ran on build machines where that path resolved. Fixed in v1.0.2 (grammar embedded into the
> binary); see `docs/release_notes/v1.0.2.md`.

The first production-honest build: the same engine and rules, now with its claims aligned to its evidence — cross-platform determinism proven and published, the integration surface verified end-to-end, and the public docs matching what actually ships.

## What changed since v1.0.0

- **Duplicate input fields are now rejected** (`AX_ERR_DUPLICATE_FIELD`), replacing the previous silent last-write-wins. Last-write-wins was order-dependent — a determinism hazard — and rejecting it brings evaluation into compliance with the engine's canonicalization contract. *Integrator note: if you previously passed duplicate field names, you now receive a defined error instead of a silently-chosen value.* The error enum stays append-only; existing codes are unchanged.
- **Language version label corrected to `0.9`** — the implementation tag for the v1.0 language spec, now consistent with the compiler and bytecode. The product version is `1.0.1`; the language-implementation tag is `0.9` (see `docs/compatibility_matrix.md`).
- **The bundle is now self-sufficient.** The documented quickstart and every shipped example run from the bundle alone, verified against the shipped binaries on both Linux x64 and Windows x64.
- **New examples:** forward-compatible unknown-error-code handling, malformed-bytecode rejection, and a producer-side replay-proof example. Replay is now exercised end-to-end in CI.

## Cross-platform determinism — published, not asserted

The same compiled bytecode and inputs produce **byte-for-byte identical** decision output on Windows x64 and Linux x64. The comparison reports and the bundles that produced them are committed in the public repo under `reports/determinism_compare_v1/` — recompute the hashes yourself. Cross-platform reproducibility is proven today; independent multi-host reproducibility is honest future work.

## Compatibility

v1.0.1 is a patch release. The C ABI is backward-compatible — `AX_ERR_DUPLICATE_FIELD` was appended; existing codes and signatures are unchanged. The single behavior change to be aware of is duplicate-field rejection, noted above.

## Verify this build

- Validate the bundle: `sha256sum -c manifests/HASHES.txt`
- Confirm the compiler: `ruledslc --version` reports `RuleDSL/1.0.1` with `lang=0.9`
- Run the smoke path: compile → verify → evaluate (`docs/quickstart.md`)
